「Webコンテンツの作成」の成果

本テーマの学習目的

まずこのテーマでは、HTML,、SS、JavaScriptの基本を学び、これらを組み合わせて Web ページを構成する方法を学ぶ。

学習目的は

  • HTML によって Web ページの構造を記述する基本的な方法がわかる.
  • CSS によって Web ページの見た目を設定する基本的な方法がわかる.
  • JavaScript によって Web ページのインタラクティブな処理を記述する基本的な方法がわかる.
  • HTML, CSS, JavaScript の構文リファレンスから，構文の書式や使用方法の情報を入手できる.
  • HTML, CSS, JavaScript, 画像・音声・動画ファイルを組み合わせて簡単な Web ページが作成できる.

プロジェクト課題

概要：

  1. グルメ情報についてページ上に設置されたプルダウンメニューからユーザーが調べたいお店の食べ物のジャンルを選択するとWWW のサーバへ検索し、検索結果を表示するページを作成する。検索内容に応じてお店の情報が出てきて、検索結果の合計と出てきたお店の情報が何軒目かを表示するようにしている。前回の検索結果は消える。コンソールにはジャンルごとに振り分けた番号が表示されている。お店のジャンルは17種類、お店の情報については8項目ある。
プロジェクト課題で作成したページ:project.html

数当てゲーム

概要：

  1. 数当てゲームは3回に分けてプログラムを完成させた。最初は、乱数で1〜10の値を1つ決めて、「答え」とし、予想は”4”で固定する。それを自動的に3回まで予想し、予想の4と「答え」があっているなら「正解です。おめでとう！」と表示し、2回目以降は「答えは ◯ でした．すでにゲームは終わっています」というのを全てコンソール上に表示する。
  2. 次は結果をページ上に表示できるよう改良した。
  3. 最後、ページ上にテキストボックスとボタンを設置し、ユーザは答えを予想して、数を入力することができボタンを押すと答えと予想が合っているかの判定ができるようになる。1回目にはなかったユーザの入力が外れた場合は、入力よりも答えが大きいか小さいかも表示される。3回予想が外れる、1度当たったあとまた入力すればゲームが終わっていることを表示する。
数当てゲームのページ:kazuate.html

各授業回の学習事項

第1回 WWWとHTML

第1回目はHTML について学んだ。HTMLとはWeb ページの構造を記述するための書式で、HTML文書を作ることによってWebページに掲載するコンテンツと、ページの基本的なレイアウトを設定できる。今回のHTMLの要素は画像の埋め込み方とハイパーリンクの設置の仕方、色々なテキストの要素について学んだ。

演習の成果：演習1-4:HTMLのさまざまな要素

第2回 CSSによるWebページの装飾

第2回目はCSSについて学んだ。CSSとはページの装飾をするものでHTMLの後に書く編集専用ファイルのようなもの。装飾は数多くあるのでMmdn Web docsで探すと便利。Fire boxでweb開発ツールが使えてそのページのHTMLとCSSの設定状況が見れる。

演習の成果：演習2-3:ウェブブラウザ説明ページの装飾

第3回 JavaScriptの基本

第3回目はJavaScript について学んだ。JavaScriptはWebページ上で複雑な機能を実現するプログラミング言語で、Web ページを開いた直後や、ボタンを押したときにJavaScriptプログラムが実行される。ページの自動更新やアニメーションなど動きのあるページを実現するためにJavaScriptプログラムを使う。

> 演習の成果：演習3-6:オブジェクトの参照

第4回 DOMとJavaScript

第4回目ではDOMについて学んだ。DOMとはWebコンテンツを表現するデータ構造のこと。DOMを使用することによってJavaScript などのプログラムによって、Web コンテンツの内容を読み取ったり、コンテンツに変更を加えたりする処理が書けるようになる。やり方としてはFirefoxでhtmlを開いてからウェブ開発ツールのコンソールからdocument.querySelectorで書き加えたい文の箇所を探し、document.createElementなどで文を追加していったりする。できたらコードをjsに追加する。

演習の成果：演習4-2〜4-5:オブジェクトの参照

第5回 イベント処理

第5回ではイベント処理について学んだ。イベントは検索ボタンなどを押したりエラーが発生したときなどのWebページ上で起こった動作や出来事のこと。イベント処理の仕方はイベントハンドラの登録（何が発生したら）とイベントハンドラの定義（イベントが発生した時どういう処理をするか）か必要である。

演習の成果：演習5-3:オブジェクトの参照

工夫点，苦心点，強調点，感想

情報メディア実験のWebコンテンツを学んで、Webの検索結果の表示の仕方や文字のレイアウトがこんなにも大変だとは思ってもいませんでした。HTML文書ではCSSやJavaScriptに比べればそこまでの苦労は感じませんでした。ですが色々な要素があって「こんなふうに表示させたい」と思い浮かべ新しい要素を調べてもうまく使いこなせなかったり、URLの貼り方が最初は手間取った記憶があります。CSSでの苦労した点は、HTMLで使った要素の種類を同じにしてしまったものが多かったので（全てのタイトルの要素をh3にしているなど）CSSでさらに装飾をしようとしたときもう一度HTMLで属性を付け足す作業をしたところです。同じ装飾でも良いところをひとつひとつ属性をつけCSSで装飾をしてしまったのでとても時間がかかりました。時間が経ち装飾を変えたいときもどれがどれだか分からず混乱す流ことが多かったです。もっとやりたいレイアウト像が固まっていれば非効率的なやり方をせずにできたと思うので計画を立ててから作成を始めることが大事だと思いました。コマンドだけで文字のシャドウやマージン、グラデーションまでつけることができたり、他のプログラミングは結果が文字で出てくるだけのものなので、プログラミングで動きのあるものが作れて新鮮で面白かったです。そしてHTML, CSS, JavaScriptを学んで一番苦労したのがJavaScriptでした。プログラムを書くのが苦手なので今まで習ったプログラミングとは少し違った繰り返し構文のlet-of構文やletの使い方の理解にかなり時間がかかり大変でした。プロジェクト課題のオブジェクトからデータを取り出す作業やDOMを使ってページ上に表示する作業も途中友達に教わりながら作りました。さらに前に書いた通り属性をたくさんつけてしまっていたので、グルメ情報の一つの項目に対して10文くらい書いていてとても大変な作業でした。そのためJavaScriptにおいてはあまりオリジナリティを出したりはできませんでした。ですがCSSでの装飾は授業で習ったものだけでなく他のサイトも活用し色々な装飾をしました。今回Webデザインではプログラムを作れることも重要だということがよくわかりました。Webコンテンツの授業で学んだことはたくさんありますが将来Webデザイン系の職業につきたいと思っているので忘れないようにときどき自分が作ったプログラムを見返したりしようと思います。プロジェクト課題に関してはプログラムが一人ではどうしても作れず友達にやり方を教わった部分があります。ですが一人で作れるのが普通なのは知っていますが数当てゲームは自分の力で作成できたので良かったと思っています。これからも今回の授業のようにプログラムを作ることはたくさんあると思うのであと2年の間にプログラムを書けるようにしいていきたいです。
